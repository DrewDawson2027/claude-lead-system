## Highest-Priority Findings

1. `mcp-coordinator/lib/workers.js` + `mcp-coordinator/lib/platform/common.js`
   - Background worker launch is effectively Unix-only because background mode always shells through `sh -c`, even when the generated script is Windows-specific.
   - Impact: advertised cross-platform/background execution is unreliable on Windows.
   - Severity: `🔴 Broken/Critical`

2. `mcp-coordinator/lib/workers.js` + `mcp-coordinator/lib/team-dispatch.js`
   - Interactive worker instructions tell dispatched workers to update the task board using the worker task ID (`W...`) instead of the team task ID (`T...`).
   - Impact: team board state can remain stale after work is completed.
   - Severity: `⚠️ Needs Work`

3. `sidecar/core/terminal-health.js`
   - Dead-shell detection expects PID files to contain JSON, but coordinator/runtime writers emit plain text PIDs.
   - Impact: dead workers can be missed in real deployments.
   - Severity: `⚠️ Needs Work`

4. `mcp-coordinator/lib/team-tasking.js` + `sidecar/server/snapshot-builder.js` + `sidecar/server/routes/teams.ts`
   - Sidecar snapshots expose only pending team tasks because the builder normalizes `task_queue` rather than the full task corpus.
   - Impact: `/teams/:team_name` observability is incomplete and can mislead operators.
   - Severity: `⚠️ Needs Work`

5. `sidecar/core/policy-engine.js` + `sidecar/server/runtime/maintenance.ts`
   - Priority aging is implemented as in-memory mutation on snapshot tasks, but not persisted back to coordinator task files.
   - Impact: queue-aging capability is mostly claimed, not mechanically enforced.
   - Severity: `⚠️ Needs Work`
